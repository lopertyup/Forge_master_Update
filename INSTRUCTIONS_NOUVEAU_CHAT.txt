## 2. Décisions importantes prises pendant le chantier

1. **`data/` est l'unique source de vérité au runtime**. `2026_04_22/` est juste un dump
   de patch pour comparer/extraire — pas lu directement par le code.
2. **`WeaponLibrary.AttackDuration` vaut 1.5 dans le JSON pour TOUTES les armes**, mais
   le wiki UI utilise des valeurs plus petites par arme (~1.10 pour Siren Song).
   Le code `BreakpointTables.tsx` consomme `weaponAttackDuration` comme paramètre, pas
   depuis le JSON directement. Donc le calculateur doit accepter un override.
3. **`PetBalancingLibrary` n'a que 3 entrées** (Balanced/Damage/Health), pas 18.
   Le scaling de rarity vit dans `PetUpgradeLibrary.LevelInfo[]`.
4. **L'attack speed est un MULTIPLIER** : `attackSpeedMultiplier = 1 + Σ(+X% AS / 100)`.
   Discrétisation en 0.1s au niveau de chaque phase (windup, recovery, double-delay)
   APRÈS division par le multiplier.
5. **Cycle minimum = 0.4s** (hard cap) ; **overhead +0.2s** ajouté à chaque cycle.
6. **Skin weapons (Age 999/1000)** : 12 nouvelles armes ajoutées au patch. On les a
   importées dans `WeaponLibrary.json` et `ItemBalancingLibrary.json`. Les noms in-game
   sont à renseigner manuellement plus tard (TODO).

## 4. Détails techniques utiles pour la suite

### 4.1 Formule complète attack speed (référence rapide)

```python
# Inputs
base_windup    = WeaponLibrary[key]["WindupTime"]
base_duration  = WeaponLibrary[key]["AttackDuration"]   # 1.5 dans le JSON,
                                                          # mais wiki UI override
speed_mult     = max(0.1, 1 + attack_speed_pct / 100)
d_chance       = min(double_damage_pct / 100, 1.0)

# Stepped phases (0.1s discrétisation)
base_recovery  = max(0, base_duration - base_windup)
sw = floor(base_windup   / speed_mult * 10) / 10
sr = floor(base_recovery / speed_mult * 10) / 10
cycle = max(0.4, sw + sr + 0.2)

# Double Attack
sd = floor(0.25 / speed_mult * 10) / 10
double_cycle = cycle + sd

# Weighted APS (APS réel pondéré par chance double)
avg_cycle = (1 - d_chance) * cycle + d_chance * double_cycle
weighted_aps = (1 + d_chance) / avg_cycle

# Final DPS
crit_mult = 1 + min(crit_chance, 1) * (crit_damage - 1)
real_dps = total_damage * weighted_aps * crit_mult
```

### 4.2 Constantes (depuis statEngine.ts)

| Constante | Valeur | Description |
|---|---|---|
| `FRAME_QUANT_S` | 0.1 | Discrétisation par frame |
| `MIN_CYCLE_S` | 0.4 | Cycle minimum (hard cap) |
| `CYCLE_OVERHEAD_S` | 0.2 | Overhead fixe par cycle |
| `DOUBLE_HIT_BASE_DELAY_S` | 0.25 | Délai base double attack |
| `MIN_SPEED_MULT` | 0.1 | Floor du multiplier de vitesse |
| `PlayerBaseDamage` | 10.0 | Base damage joueur |
| `PlayerBaseHealth` | 80.0 | Base health joueur |
| `PlayerMeleeDamageMultiplier` | 1.6 | Multiplier melee × base damage |
| `PlayerBaseCritDamage` | 0.20 | +20% crit damage de base |
| `LevelScalingBase` | 1.01 | Scaling par niveau d'item |
| `ItemBaseMaxLevel` | 98 | Cap level d'item (peut monter via TechTree) |

### 4.3 Conventions de nommage des sprites

Tous les noms doivent avoir :
- **espace entre les mots** ("Saber Tooth", "Cannon Barrage", "Brown Horse")
- **majuscule en première lettre de chaque mot**
- correspondance EXACTE avec les filenames PNG dans `data/icons/`

Les `Auto*Mapping.json` files sont la source de vérité pour ces noms.
